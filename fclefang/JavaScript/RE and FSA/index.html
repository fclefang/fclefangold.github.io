RE and FSA

  • NLP
RE-FSA-FST

1.正则表达式

正则表达式regular expression简称RE是一种用于描述文本搜索符号串的语言。最初由Kleene开发是专用语言中用于描述符号串的简单类别公式，从形式上来说是用来刻画符号串集合的一个代数表述。可以用来描述搜索符号串，也可以用于以形式的方法定义一种语言。

  • 正则表达式是区分大小写的，表达式前后加斜线‘/’，内部有‘[]’的字符串表达式是表示字符的析取匹配，比如/[wW]oodchuck/则会匹配Woodchuck或者woodchuck。
  • 连字符‘-’可以用来表示一定范围内的任何字符，比如/[1-3]/则表示字符1,2,3中的任一个字符。
  • 脱字符‘^’，‘[]’还可以表示不出现某个单独的字符也就是如果再方括号之后又脱字符那么相应的模式就是否定的但是需要注意这种用法仅当脱字符处于开方括号之后的第一个位置才有效，在其他位置仍然表示脱字符本身。比如[^A-Z]\表示不是一个大写字母，而[e^]\表示字母e或者脱字符^。另外还可以表示一行的开始比如/^The/表示单词The只出现在一行的开始。
  • 问号’?’表示前一个字符的有或无。
  • ‘’称为“Kleene ”表示前面紧邻的字符好着正则表达式出现零次或者连续出现多次。
  • ‘+’表示前面一个或者多个字符。
  • ‘.’是一个通配符，表示任何与单个字符(回车符除外)相匹配的字符。
  • ‘_$’表示一行的结尾是空白，比如/^The dog.$/表示仅包含短语The dog的一个行。
  • ‘\b’表示词界，’\B’表示非词界，比如/\bthe\b/表示单词the，the前后不能是数字下划线或者字母。
  • 圆括号运算符‘()’，把一个模式括在圆括号内，它能像一个单独的字符来使用。比如/gupp(y|ies)/表示guppy或者guppies，这里的’|’是析取符，就是‘或’的意思。
  • 有了圆括号，计数符，序列与锚和析取符(优先级从高到低)，我们可以组合使用表示不同的内涵。另外更多的高级算符这里就不一一列举了。

2. 有限状态自动机

看着就和计算机理论中图灵机有很多类似的思想。
FSA可以分为DFSA和NFSA，值得一提的是，对于任何NFSA又存在与之对应的DFSA，正则表达式的每个基本操作都可以通过自动机模拟，就和图灵机模拟可计算函数一样。在NFSA情况下，由于在某结点处存在一种以上的选择可能性(这也就是为何我们称之为非确定有限状态机)，我们有可能做出错误的选择，面对这样的问题这里先介绍回退方法解决该问题。

  • 回退：每当我们走到这样的选择点时，我们可以做一个记号，记录下在纸带输入中的什么位置以及自动机处于什么状态。当我们确认做了错误选择时可以退回去试探其他的选择可能。该方法两个重点：在每个选择点必须记住所有选择；对于每个不同选择必须存储足够的信息以便需要时返回。当返回算法到达某个点而其进程不能再进展时算法返回到前一个选择点开始重复迭代不同路径。这里我们称自动机状态为结点或者机器状态，把纸带上位置和结点的结合体称为搜索状态。
  • 下面是用于NFSA识别的算法。ND-Recognize首先建立一个初始的搜索状态，并把它放入进程表中。NEXT函数功能是从进程表中检索一个项目(尚未进行过的选择)并且把它赋给变量current-search-state(当前搜索状态)。和DFSA识别算法类似，主要回路的首要任务也是确认纸带上的输入信息是否被自动机成功识别。这里通过ACCEPT-STATE？函数来实现，如果current-search-state既包含一个接受的机器状态也包含一个纸带结尾的指针就返回accept。否则调用GENERATE-NEW-STATE函数来生成新的可能的下一状态，所有这些创建的状态都要加到当前的进程表中。最后再对进程表中新搜索状态处理，继续选择其他没尝试过的可能性选择，若仍然无法进行选择则拒绝输入的字符串。
  • ND识别算法的任务在于提出一种办法来系统地探索自动机中所有可能的路径，从而识别正则表达式表示的符号串。如果找到了这样一条路径并且以接收状态结束那么accept该符号串否则就refuse。利用进程表可以实现这样的探索机制。ND识别这种算法称为状态空间搜索算法。在可能的路径空间内搜索问题正确的解。进程表就是这样的“空间”，算法将下一步要处理的状态置于进程表前端然后通过stack实现，这种处理顺序的策略通常被称为深度优先搜索或者后进先出LIFO策略。当然还有用队列来实现的策略又称为广度优先搜索或者先进先出策略。

3. 历史小事

FSA来源于Turing-theory，McCulloch和Pitts研制了类似FSA的神经元模型，在神经元模型基础上Kleene定义了有限自动机和正则表达式并且证明二者等价，后来Rabin和Scott提出非确定自动机并证明与确定自动机等价。

4. 有限状态转录机FST

关于语言形态学中曲折形态学和派生形态学不详细介绍，简单举个例子就像规则名词和不规则名字的复数形式以及动词的各种过去式啥的都是曲折形态学中的范畴，而派生形态就比如一个动词加上less变成形容词就是派生范畴，派生形态学比曲折形态复杂多，这里面的知识不作详细介绍。下面主要介绍FST。

  • 要知道FSA是用来解决形态识别的，所谓形态识别就是判断由字母构成的输入符号串是不是合法的，当我们用FSA来表示词表以及进行形态识别之后就会面临形态剖析问题：例如‘cats’经过剖析得到输出‘cat+N+PL’，意思是cat的复数名词。而这里的剖析过程就是FST来完成的。
  • 这里FST的思想是用的双层形态学方法，将一个词分为表层和词汇层，表层是该词实际拼写情况，而词汇层则是表层上的字母序列映射之后的语素和特征的序列信息(比如cat+N+PL)。不难理解FST就是用来完成这个映射任务的。其实吧，双层形态的双层就可以看成两个纸带，纸带上存储各自层的信息，而FST也就是具有两个纸带的自动机，我们似乎也可以把它的机制理解成在一个纸带上识别形态并且在另一个纸带上生成被识别形态的语素和特征序列。
  • 然而双层模型对于多样复杂的词汇仍然有很大的局限，尤其在处理英语中经常出现的语素边界拼写变化的问题，所以改进提出了三层模型，在表层和词汇层之间加一个中间层，在表层和中间层之间引入拼写规则又称为正词法规则来解决语素边界的问题。
  • 将FST引入规则相结合对词汇进行剖析和生成下面一张图很容易让人理解层与层之间的机制。
    从图中看很好理解FST剖析以及生成的过程，这里剖析过程相对复杂，因为在剖析的时候存在歧义问题，比如foxes还可以表示动词的意思，出现这样的歧义FST是没有能力解决的，关于歧义消解会另外总结。
  • 建立词表加规则的FST是形态剖析的标准算法，当然还存在一些简单的额算法并没有标准算法所需的大规模联机词表的要求这样的简单算法可以用于完成信息检索任务，比如简单有效的Porter词干处理器算法，并不需要精确的形态结构分析。
  • 另外存在FST的自动编译程序，该编译程序对于任何简单的重写规则都能造出一个FST出来。
