2. VMware Tools

如果未装载 VMware Tools 虚拟 CD-ROM 映像，请装载 CD-ROM 驱动器。

  • 如果装载点目录尚不存在，请创建该目录:mkdir /mnt/cdrom
    某些 Linux 发行版使用不同的装载点名称。例如，某些发行版上的装载点是:
    /media/VMware Tools而不是/mnt/cdrom。请修改该命令以反映您的发行版使用的约定。

  • 装载 CD-ROM 驱动器:mount /dev/cdrom /mnt/cdrom
    某些 Linux 发行版使用不同的设备名称，或者以不同的方式组织 /dev 目录。如果 CD-ROM 驱动器不是 /dev/cdrom 或 CD-ROM 装载点不是/mnt/cdrom，则必须修改该命令以反映您的发行版使用的约定。

    3. 本机与linux虚拟机的互通

    因为知识的缺乏，在网络配置方面有点白痴，所以跟着老师视频里的教程进行试验时出现了好几个小漏洞，虽然花了几个小时，但最后总算实现了的本机Xshell操作台实现对Linux虚拟机的控制。

  • ‘：wq’:文件保存并退出

  • ‘bc’:计算器
  • ‘sync’:数据同步写入磁盘
  • ‘shutdown’:此外，需要注意的是，时间参数请务必加入指令中，否则shutdown会自动跳到 run-level 1
  • 其他一些常见的可以参考文哥博客
    实现互通的解决思路：其实这次搞完主机和虚拟机互通之后，对于老师的讲解其实有一点疑问。我自己是觉得主机和虚拟机借助于共同的网关192.168.1.1.

首先：dos界面下ipconfig命令取得当前主机的IP地址以及网关地址。
然后：虚拟机设置为桥接模式，setup进入网络配置eth0，配置一个合理的IP地址，以及相同的网关地址，当然DNS以及子网掩码也需要配置好。当然也可以自己对eth0进行ifconfig配置。
1、配置网卡eth0静态IP。
2、将其中的ONBOOT设置为no
3、重新启动机器
4、使用ifconfig eth0 up来激活网卡的，ifconfig 查看eth0，eth0的信息并有显示eth0的静态ip。
5、改用ifup eth0 来激活网卡，ifconfig查看eth0，此时的eth0的信息中静态IP就被启用。
6、如果将ONBOOT设置为yes，重新启动后，ifconfig up与ifup的效果是一样的。
在网百度上搜了一下ifconfig up 与ifup的区别，只查到说ifconfig up 会回调ifup脚本。
使用ifup和ifdown脚本“启用”或“禁用“特定的网络接口比直接使用
ifconfig命令要好一些，这是因为脚本可以认真查看并处理与某一个
特定接口有关的所有参数和相关功能。
ifup和ifdown这两个脚本通常放在/etc/sysconfig/network-scripts/或者/etc/network/下面。
最后：利用Xshell新建一个连接，配置好虚拟机eth0的IP地址就可以连接上虚拟机实现主机控制虚拟机的实验。
PS：vi /etc/sysconfig/network-scripts/ifcfg-eth0配置网卡：ONBOOT=yes这句是什么含义？
解答：ONBOOT是指明在系统启动时是否激活网卡，只有在激活状态的网卡才能去连接网络，进行网络通讯。
