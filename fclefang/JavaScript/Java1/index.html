java1
  1. 环境变量：path以及classpath的配置，当配置可执行文件时用path，当配置java文件时用classpath.
    path配置完成，操作系统先查找当前目录下然后再查找环境变量path配置的路径，而classpath配置后操作系统的查找顺序则相反。
    当用set path进行临时配置时，结束后不会影响本机原有配置。
  2. 重载overload
    在同一个类中，允许存在一个以上的同名函数，只要它们的参数个数或者参数类型不同即可。
  3. 类和对象
    如下一小段代码：

Car c = new Car();这样就产生了栈内存和堆内存。这句代码让栈内存中就产生了Car类型的变量c，而在堆中就产生了实体(包含实体的一些内容属性)，实体所在的地址就赋值给了c，所以变量c就指向了该实体。如果再执行Car c1 = new Car();就会在堆中产生一个新的实体，栈中新的变量c1指向该新实体，而并不是指向之前的实体。

  • 成员变量和局部变量：成员变量作用域整个类中，而局部变量作用于函数或者语句中。成员变量存在于堆内存中，而局部变量存在于栈内存中。
  • 构造函数：对象一建立就会调用与之对应的构造函数，可以用于给对象进行初始化。可以通过构造函数的参数不同给对象进行不同的初始化。如果某个构造函数加上private属性私有化那么Person p = new Person("Lisi");就不能创建该对象。
  • 构造代码块：给对象进行初始化，对象一建立就执行，而且优先于构造函数执行。给所有对象统一初始化而构造函数则是给相对应的对象初始化。所以可以借助于构造代码块给对象包含的一些共性的特征进行初始化。
  • this：看上去用于区分局部变量和全局变量同名的情况。代表所在函数所属对象的引用，也就是说哪个对象调用this所在的函数，this就代表哪个对象。构造函数间调用只能用this语句！只能定义在构造函数的第一行，因为初始化要先执行。
  • static：比如当我们在Person类中static String country = "cn";当使用静态修饰时，那么这个country就不随着其他属性存在于堆中，而是成为一个共享属性和方法一起存在于方法区(共享区，数据区)中。可以被类名直接调用，静态方法中不可以定义this和super关键词，因为静态方法优先于对象存在。
